На нашем [конструкторе](https://quescha.com) вы сможете собрать практически любого чат-бота, от самых простых пошаговых до суперсложных, состоящих из нескольких чат-ботов, между которыми вы сможете передавать данные. Кроме того, вы сможете использовать в схемах своих ботов Гугл Таблицы, CRM, платежные системы, умные ссылки, нейросеть ChatGPT, обращаться к API внешних ресурсов, принимать вебхук, Web App Telegram, Google Календарь, Google диск, использовать Javascript код в чат-боте.

<figure><img src=".gitbook/assets/Схема Quescha (1600).png" alt=""><figcaption></figcaption></figure>

Конструктор позволяет передавать сообщения от пользователя к пользователю, пересылать файлы, делать авторизацию в закрытые разделы, создавать платные подписные модели пользования информацией, заполнять по шаблону и отправлять договора, сметы, чеки, получать уведомления о забытом шаге, делать задержки, серии сообщений, заменять одно сообщение другим, запрашивать телефон и геолокацию, отправлять информацию менеджерам и многое другое.

Использование конструктора не требует навыков программирования, однако следует разобраться в его особенностях, чтобы перед вами раскрылась вся его мощь. По сути это программирование нового типа, так популярного сегодня "no code" программирования. Строить алгоритмы взаимодействия с клиентами и сотрудниками, а также работать с данными внутри и вне системы, вот что такое настройка чат-ботов сегодня.

Для запуска простых автоворонок, например постинга сообщений с определенным интервалом, вам мало с чем придется разбираться.

Если вам потребуется сделать минимальный пошаговый сбор данных, то придется использовать режим сценария для сообщения в цепочке.

А если вы хотите делать сложнофункциональных чат-ботов с гугл-таблицами, платежами, CRM и т.д., то без сценариев действий вам не обойтись. Сценарий действий также используется для отправки сообщений в бот к нужному пользователю.

![](<.gitbook/assets/Автоворонки (4).png>)



После регистрации в конструкторе вам предоставляется 7 дней полного функционала. Число подписчиков не должно превысить 100.

Далее вы можете использовать **бесплатный тариф** конструктора бессрочно, пока вы учитесь с ним работать. Этот тариф включается автоматически, если на балансе не будет достаточно средств для списания выбранного тарифа. Если у вас уже был тариф выше Старта, то убедитесь, что он соответствует тарифу Старт по количеству лендингов (1). Также количество подписчиков не должно превышать 100.
