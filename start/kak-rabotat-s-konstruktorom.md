# Как работать с конструктором

Конструктор выполняет заданную вами логику и отправляет сообщения в каналы. Канал - это мессенджер (а также бот-лендинг/виджет для сайта). Первое, что необходимо сделать, это добавить своего бота на странице Каналы.

Чат-бот состоит из логики его работы и каналов. Логика работы чат-бота это автоворонка.

Автоворонка, в свою очередь состоит из цепочек сообщений. **Цепочки сообщений** нужны для сегментации вашей аудитории и для выполнения различных сценариев в одном чат-боте. Главная цепочка, как правило, запускается при подписке, остальные запускаются при добавлении тегов.

Ваш чат-бот может быть авторассылкой, сценарием или комбинацией авторассылки и сценария.

**Авторассылка** это периодическая отправка сообщений подписчикам. Авторассылки не предполагают сбора данных от подписчиков. Подписчик может запустить другую цепочку по кнопке. Таким образом, чат-бот сегментирует аудиторию в соответствии с их интересами.

**Сценарий** дает возможность собирать данные от подписчиков, работать с данными, работать с интеграциями, платежами и т.д. Каждый сценарий состоит из блоков. В процессе взаимодействия с подписчиком сценарий движется по шагам от блока к блоку.

В сценариях могут быть использованы **сценарии действий**. Они дополнительно расширяют возможности сценариев. Каждый блок может вызывать тот или иной сценарий действий для необходимой работы. Чтобы отобразить сообщение, которое возвращает сценарий действий, в блоке используется шаблон \[Actions].

В сценарии действий вы можете отправить собранные данные в бот администратора. [Подробнее](../privyazka-sotrudnikov-otpravka-zayavok.md).

Перезапустить бот в любой момент вы можете командой /start
